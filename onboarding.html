Lets pick a set of rules...

How thorough would you like to be?

You can change, or include additional settings, later in your profile page.

STARTUP

No crazy rules. Just partial reviews, tickets and comments in a super cool way.

  • SELECT
  • Partial reviews
    Check the code while it's being written.
  • No opened issues
    No merging is allowed until all issues are closed.

STANDARD

Recommended by Gitcolony!

MOST USED!
  • SELECT
  • Partial reviews
    Check the code while it's being written.
  • No opened issues
    No merging is allowed until all issues are closed.
  • Voting
    Merging can be done only after 1 developer voted for it.

POWER

A clockwork tailored to your team's very needs.
Kick-off your OCD!

  • SELECT
  • Partial reviews
    Check the code while it's being written.
  • No opened issues
    No merging is allowed until all issues are closed.
  • Qualified voting
    Merging can be done only after 1 developer and 1 QA member voted for it.
  • Automatic updates
    When the destination branch gets new commits, all opened PRs pointing to it get actualized with those commits.
  • Automatic PR
    A PR is created every time a new branch is added to the repo.
  • Linked PRs
    All linked PRs must comply with the rules in order to be merged.
